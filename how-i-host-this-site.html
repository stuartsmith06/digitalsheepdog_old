How I host this site

Update:  For several reasons, I have "re-homed" this blog site and will be posting a new article about this soon.

If you’re reading this post, I suspect you were searching specifically on how to host a static website in AWS S3.  This guide will also show you how to set up a static content site with the following characteristics:

  • Site content is hosted in an S3 bucket that is not publicly accessible
  • Site runs behind a CloudFront distribution
  • Site has an AWS-managed TLS certificate with all traffic encrypted

Strictly speaking, the screenshots below aren’t showing this site (Digital Sheepdog), but how I set up my other blog site, Apologetics Geek.  But the settings are similar, and these instructions will work for your site as well.

All information and screenshots are accurate as of December 2020.  As you’re likely aware, Amazon is constantly making feature updates and changes to the AWS console.  In the future some of these screenshots may be different.  But conceptually, this should still work.  This guide assumes you already have an AWS account and are generally familiar with how to navigate around the AWS Console.

There are five steps that you have to follow in order. 

  1. Register (or transfer) your website’s domain name with Route53
  2. Create a S3 bucket to store your site’s content
  3. Request a TLS Certificate in Amazon Certificate Manager
  4. Create a CloudFront distribution
  5. Add an DNS Alias record to your CloudFront distribution in Route53

A quick note about cost:  This solution is very inexpensive, but it’s not free.  For example, registering a .com domain with AWS is $12 per year.  S3 pricing is very cheap; my total S3 bill is about $1 per month.  Route53 is $1 per hosted zone., plus some minor DNS query cost.  

Step 1:  Register or transfer a domain name

From the AWS Console, go to Route53. 

Under Domains, click “Registered Domains” (or if you’re transferring a domain, click the “Transfer Domain” button—follow the instructions there on your own).

Select a domain name, and go through the registration process, which will include entering your Registrant Contact information, and actually paying for the domain name.

Once your domain name is set up, you’ll see it in your list of Registered domains in Route53.

 

Step 2:  Create an S3 bucket to store your site’s content

From the AWS Console, go to S3.  Under “Buckets”, click “Create bucket”.

The bucket name should match the FQDN of your website.

The default setting is “Block all public access”.  Leave this box checked!

You can leave Bucket Versioning and Default Encryption disabled. All other settings should be default/disabled. 

Click “Create bucket”, and you’re done.

You'll see your newly created bucket. Note, "Bucket and objects not public" is what we want!

Next, add some files, at minimum a basic "index.html” web page to start.  You can use a static website generator like Publii (https://getpublii.com/), or one of many others out there.

Upload your “index.html” file (or upload your entire site if you’ve already created it).  

 

Step 3:  Create a TLS Certificate in Certificate Manager

From the AWS Console, got to Certificate Manager.

Click “Request a certificate” and chose the “Request a public certificate” option.

Add domain name that you own.  While “wildcard” certificates are allowed, it’s generally a best practice to specify the names you want on the certificate.

Click next, then select “DNS validation” as the validation method.

Enter any tags you wish, then click “Review”, then click “Confirm and request”.

You’ll see your new certificate in a “Pending validation” state.

Important:  Don’t click “Continue” yet.  AWS has to validate that you own the domain name for which you’re requesting the TLS certificate.  Since we chose the “DNS Validation” option, we’re going to add a custom validation string to our Route53 domain record.

Fortunately, AWS makes this very easy!

And you’ll see an option to “Create record in Route 53”.  Click that button for all domain names associated with your certificate.

You'll get a pop-up with the options (don't change anything), and click "Create."

If you go over to Route53 and you’ll see the new CNAME records associated with your domain record:

The AWS Certificate Manager will do a DNS lookup for these entries, and once found, this will valdate your ownership of the domain.

Now go back to Cert Manager and click “Continue”.

Eventually (this can take several minutes), you’ll see your new TLS Certificate’s status changed to “Issued”.

 

Step 4:  Create CloudFront distribution.

From the AWS Console, go to CloudFront.

Click “Create Distribution”

We’ll be creating a “Web” distribution, so under that section, click “Get Started”. 

In the Origin Domain Name:  Select your newly created S3 bucket from the dropdown as the Origin Path.

Here are the important settings you need to make:

Restrict Bucket Access:  Yes

Origin Access identity:  Create a New Identity (easiest)

Grant Read Permissions on Bucket:  Yes, Update Bucket Policy

Default Cache Behavior Settings:  Redirect HTTP to HTTPS

Distribution Settings:

Alternate Domain Names: (here, enter your sites FQDN and enter “www.{fqdn}”

Custom SSL Certificate:  Select the TLS cert you created in Certificate Manager above

Important > Default Root Object:  index.html

Standard Logging:  (optional, but I recommend logging.  You’ll need to set up a separate S3 bucket for CloudFront logs)

Uncheck “Enable IPv6” (optional)

Click “Create Distribution”

Click back on “Distributions”, you’ll see Your new distribution with an Enabled State, and Status “In Progress”

Give it a few minutes, and eventually you will see “Deployed” in the Status column. 

Go ahead and select the Domain Name and copy it.

(In my example, the CF domain name is: d11skgzpmxnc37.cloudfront.net)

Once your CF distribution status is “Deployed”, open a new browser window and paste the default domain name of your distribution.  You should see your “index” page load.

 

Step 5: Create a DNS Alias record to your CloudFront distribution

From the AWS Console, go to Route53.  From the Dashboard, click the link under DNS Management that says “Hosted zones.”

Then click the appropriate domain name.

Click “Create Record Set” with the following attributes.

Type: A – IPv4 address

Alias:  Yes

Alias Target:  paste your CF Distribution name (d11skgzpmxnc37.cloudfront.net)

Do this for “www” as well if desired.

You’ll end up with two DNS “A” records that point to the Cloud Front distribution.

It will take several minutes for DNS to update, but once that does, when you type in your domain name in your web browser, you should see your “index” page.

With your TLS Certificate:

Congratulations!  You now have a very low-cost, static website hosted in AWS.
